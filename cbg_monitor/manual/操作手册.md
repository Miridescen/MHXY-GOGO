# 藏宝阁全服比价 · 爬取与导入操作手册

> 目标：**任何一台电脑**（不依赖 Claude）都能自己爬数据、导入服务器，网站自动更新。

## 总览（一步搞定）

```
浏览器爬取(任意电脑Chrome)
  登录藏宝阁 → F12控制台贴脚本 → 进度条跑完
  → 自动 POST 入库(HTTPS令牌接口) → 网站自动更新   ✅ 不用下载、不用命令
```

- 数据库在服务器：`/opt/cbg-data/prices.db`（SQLite）
- 网站：**https://43-106-131-65.nip.io:8090/** （后端自动读库，入库后即刷新）
- 导入接口：`POST https://43-106-131-65.nip.io:8090/api/ingest`（带令牌，脚本里已配好）
- 一天爬 1~2 次即可；同一天重复入库=覆盖更新当天价。

## 一次性准备（换新电脑时）

只需要两样：
1. **Chrome 浏览器** + 一个**藏宝阁账号**（网易通行证，能登录 xyq.cbg.163.com）。
2. **爬取脚本** `crawl_console.js`（在项目 `cbg_monitor/manual/` 里，或让人发你）。

> 入库走 HTTPS 接口自动完成，**不再需要 SSH/scp**。接口地址已内置；
> **令牌**首次运行脚本时会弹窗让你输入一次（向管理员索取），存在本浏览器、之后免输。
> （只有自动入库失败时，才用到附录 D 的手动 SSH 备用法。）

## 爬数据 + 自动入库（浏览器一步，约 30~40 分钟）

1. Chrome 登录藏宝阁，打开**召唤兽搜索**页：
   `https://xyq.cbg.163.com/cgi-bin/equipquery.py?act=show_overall_search_pet`
   （登录要走到能正常搜索的状态：网易账号 + 选角色「进入」）
2. 按 **F12** → 切到 **Console（控制台）** 标签。
3. 打开 `crawl_console.js`，**全选复制**，粘贴进控制台，回车。
4. 页面**顶部出现蓝色进度条**，数字往上涨（`爬取中 X/808`）。
   - ⚠️ **把这个标签页放在前台、别切走、别关**（后台会被浏览器限速一倍）。
5. 进度条跑完会**自动入库**，显示绿色「✅ 已自动入库 N 条」。**完事，不用再做任何操作。**

**进度条颜色：**
| 颜色 | 含义 | 你做什么 |
|---|---|---|
| 🔵 蓝 | 爬取中 / 自动入库中 | 等着，保持前台 |
| 🟢 绿 「已自动入库」 | 全部完成并入库 | ✅ 完事 |
| 🟠 橙 「自动入库失败」 | 接口没连上 | 点橙条下载 CSV，按**附录 D**手动导 |
| 🔴 红 · 验证码 | 触发人机验证 | 在藏宝阁随便搜一次、手动解验证码，再**重新粘贴脚本**跑 |
| 🔴 红 · 登录过期 | 掉登录 | 重新登录藏宝阁，再重新粘脚本 |

## 附录 A：加新物品（爬别的召唤兽）

编辑 `crawl_console.js` 顶部的 `ITEMS` 数组，加一行 `['物品名','类型ID,类型ID']`。

**查类型ID**：在召唤兽搜索页控制台跑（把"广目"换成你要的名字）：
```js
Object.entries(window.SaleablePetNameInfo).filter(([id,n]) => n.indexOf('广目') !== -1)
```
- 输出里**名字不带「变异」**的那两个 ID 就是正常版（如广目巡守 `102337,102338`）。
- 带「变异」前缀的（一般是 +500 的 ID）是变异版，**不要**。

导入后，新物品默认按「宝宝」品类显示。要归到别的品类：
```bash
ssh root@43.106.131.65 "sqlite3 /opt/cbg-data/prices.db \"UPDATE item SET category_id=3 WHERE name='灵饰名'\""
# 品类码: 1宝宝 2装备 3灵饰 4内丹 5锦衣 6材料
```

## 附录 B：排错

- **进度条变红·验证码**：并发或太频繁触发了网易易盾。本脚本已是单线程慢速，正常很少触发；真遇到就手动解一次验证码后重跑。**绝不要把脚本里的间隔调小或改成并发**（封号风险）。
- **掉登录**：重新登录藏宝阁再跑。
- **进度条变橙·自动入库失败**：接口没连上（服务器挂了/网络/令牌错）。点橙条下载 CSV，走**附录 D** 手动导。
- **网站没更新**：后端有 60 秒缓存，等 1 分钟刷新；或 `ssh root@43.106.131.65 'systemctl restart mhxy-api'`。
- **证书/HTTPS 报错**：访问要用 `https://43-106-131-65.nip.io:8090/`（不要用 IP，IP 证书不匹配）。续期是自动的；手动续期 `ssh root@43.106.131.65 'certbot renew'`。

## 附录 D：手动导入（仅当自动入库失败时）

进度条变橙时点它下载 `xunshou_YYYYMMDD.csv`，然后终端（需要 SSH 到服务器，输 root 密码）：

```bash
scp ~/Downloads/xunshou_20260625.csv root@43.106.131.65:/opt/cbg-data/runs/
ssh root@43.106.131.65 'python3 /opt/cbg-data/ingest_combined.py /opt/cbg-data/runs/xunshou_20260625.csv'
```

## 附录 C：服务器速查

| 东西 | 位置 / 命令 |
|---|---|
| 数据库 | `/opt/cbg-data/prices.db` |
| 导入脚本 | `/opt/cbg-data/ingest_combined.py` |
| 上传目录 | `/opt/cbg-data/runs/` |
| 后端服务 | `systemctl status mhxy-api`（127.0.0.1:5002，含 INGEST_TOKEN） |
| 网站 | https://43-106-131-65.nip.io:8090/ （nginx HTTPS，前端 `/var/www/mhxy/`） |
| 导入接口 | `POST /api/ingest`（X-Token 令牌；脚本里已配） |
| 证书 | `/etc/letsencrypt/live/43-106-131-65.nip.io/`（certbot 自动续期，需 80 端口常开） |
| 查某物品某天最低价 | `sqlite3 /opt/cbg-data/prices.db "SELECT i.name,p.serverid,p.price_yuan FROM price_history p JOIN item i ON i.id=p.item_id WHERE p.run_time='2026-06-25' AND i.name='谛听' ORDER BY p.price_yuan LIMIT 10"` |
| 看所有物品 | `sqlite3 /opt/cbg-data/prices.db "SELECT * FROM item"` |

## 表结构速记

- `item(id, name, type_ids, category_id→category.id)` — 物品
- `category(id, name, icon, color_bg, color_fg)` — 品类维表(1宝宝 2装备 3灵饰 4内丹 5锦衣 6材料)
- `price_history(id, item_id, run_time日期, serverid, price_yuan, link, eid)` — 价格历史，唯一键 `(item_id,run_time,serverid)`
- `server_map(id, product, area_name, server_name, serverid, server_age)` — 区服映射(product:1时间服 2畅玩服)
