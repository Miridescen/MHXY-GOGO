# 数据库说明

SQLite 数据库：服务器 `/opt/cbg-data/prices.db`。共 4 张表。

```
关系总览：
  category ──1:N──► item ──1:N──► price_history
                                      │ serverid（非外键，靠值关联）
                                      └─────────► server_map
```

- `category`：品类维表（图标/配色的唯一真相源）
- `item`：物品（指向品类）
- `price_history`：价格历史（指向物品；按 serverid 关联区服）
- `server_map`：大区↔服务器↔serverid 映射（与 price_history 靠 serverid 值关联，未建外键）

---

## 1. `category` — 品类维表

物品品类及其展示样式（图标占位字、配色）。前端按此渲染图标方块；改品类样式只改这一张表。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | 主键 | 品类ID（固定 1~6） |
| `name` | TEXT | UNIQUE NOT NULL | 品类名 |
| `icon` | TEXT | | 图标占位汉字 |
| `color_bg` | TEXT | | 图标底色（hex） |
| `color_fg` | TEXT | | 图标前景色（hex） |

**固定取值：**

| id | name | icon | color_bg | color_fg |
|---|---|---|---|---|
| 1 | 宝宝 | 宝 | #fbeee8 | #a8351f |
| 2 | 装备 | 武 | #f3eadb | #9a7b3a |
| 3 | 灵饰 | 灵 | #e6f0ea | #3a7a5a |
| 4 | 内丹 | 丹 | #eae6f0 | #5a4a8a |
| 5 | 锦衣 | 衣 | #f6e8ea | #9a3a5a |
| 6 | 材料 | 材 | #f0ece2 | #8a7a4a |

---

## 2. `item` — 物品

要监控价格的物品。`type_ids` 是爬取时发给藏宝阁的类型ID。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | 主键 自增 | 物品ID |
| `name` | TEXT | UNIQUE NOT NULL | 物品名，如 持国巡守 |
| `type_ids` | TEXT | | 藏宝阁召唤兽类型ID（正常版，逗号分隔），如 `102242,102245` |
| `category_id` | INTEGER | → `category.id` | 品类（外键）；为空时前端按「宝宝」兜底 |

**样本：** `{id:1, name:'持国巡守', type_ids:'102242,102245', category_id:1}`

> 爬虫自动新建物品时 `category_id` 为空，需手动归类：
> `UPDATE item SET category_id=3 WHERE name='灵饰名'`

---

## 3. `price_history` — 价格历史

核心表。每物品、每服务器、每个采集日期，存一条「该服最低价」。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | 主键 自增 | |
| `item_id` | INTEGER | NOT NULL → `item.id` | 物品（外键） |
| `run_time` | TEXT | | 采集日期（**年月日**，如 `2026-06-22`） |
| `serverid` | INTEGER | | 服务器ID（关联 `server_map.serverid`） |
| `price_yuan` | REAL | | 该服该物品最低价（元） |
| `link` | TEXT | | 藏宝阁商品链接 |
| `eid` | TEXT | | 商品唯一ID |

**约束 / 索引：**
- `UNIQUE(item_id, run_time, serverid)` —— 同物品/同日/同服只一条；重复入库=覆盖更新（保留当天最新价）
- 普通索引 `idx_item_srv(item_id, serverid, run_time)`

**样本：** `{id:1, item_id:1, run_time:'2026-06-22', serverid:459, price_yuan:39.08, link:'https://xyq.cbg.163.com/equip?s=459&eid=...', eid:'...'}`

> `run_time` 只到天：一天爬两次也是覆盖同一天那条。要保留多次需另加批次字段。

---

## 4. `server_map` — 区服映射

大区→服务器→serverid 的对照表，从网易 CDN 抓取（`cbg_monitor/server_map.py` 刷新）。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | 主键 自增 | |
| `product` | INTEGER | | 服别：**1=时间服，2=畅玩服** |
| `area_name` | TEXT | | 大区，如 上海1区 |
| `server_name` | TEXT | | 服务器名，如 珍宝阁 |
| `serverid` | INTEGER | | 服务器ID（**合服会多个服务器名共用同一ID**） |
| `server_age` | INTEGER | | 开服年限（仅时间服，对齐藏宝阁搜索页）：**3=三年外(3年以上服)，2=1到3年服，1=1年内服**；NULL=未填/畅玩服 |

**约束 / 索引：**
- `UNIQUE(product, area_name, server_name)`
- 普通索引 `idx_map_sid(serverid)`

**样本：** `{id:1, product:1, area_name:'上海1区', server_name:'珍宝阁', serverid:159, server_age:NULL}`

> serverid 非唯一：一个 ID 可对应多个 server_name（合服）。所以 `price_history` 按 serverid 关联时，同 ID 的多个服名共享同一价格。

---

## 5. `role_query` — 角色搜索注册表

定义「要追踪哪些角色搜索」。条件用 JSON 存，**加新组合=插一行、加新条件维度=JSON 多一个键，表结构和爬虫都不用改**。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER | 主键 |
| `grp` | TEXT | 分组：`境界` / `锦衣`（网站分块展示用） |
| `name` | TEXT UNIQUE | 人读名，如 `飞升·3年以上`、`青花瓷·男·飞升` |
| `conditions` | TEXT(JSON) | 业务条件（给人看/网站分组） |
| `api_params` | TEXT(JSON) | 藏宝阁接口查询参数 |
| `enabled` | INTEGER | 1=启用 |

当前 1241 条，四组（角色三组 + 装备一组）：
- **境界组**（grp=境界，12 条）= 4 类别（飞升/渡劫/175级/化圣）× 3 开服年限。
  - conditions `{"类别":"飞升","开服年限":3}`；api_params `{"zhuang_zhi":"1","server_type":3}`
- **锦衣组**（grp=锦衣，280 条）= 性别(男/女) × 等级(不限/69/109/飞升/渡劫/175/化圣，共7) × 限量锦衣(20)。
  - 「不限」等级 = api_params 不带等级参数（只 limit_clothes+sex）。
  - conditions `{"锦衣":"青花瓷","性别":"男","等级":"飞升"}`；api_params `{"limit_clothes":12512,"sex":1,"zhuang_zhi":"1"}`
- **坐骑组**（grp=坐骑，28 条）= 性别 × 等级(同上7) × 限量坐骑(天使猪猪/九尾冰狐，2)。结构同锦衣，仅查询参数换成 `xiangrui_list`（值用坐骑名字）。
  - conditions `{"坐骑":"天使猪猪","性别":"男","等级":"飞升"}`；api_params `{"xiangrui_list":"天使猪猪","sex":1,"zhuang_zhi":"1"}`
- **装备组**（grp=装备，921 条）= 走「装备搜索」`search_type=overall_search_equip`。四子组（每个等级分开）：
  - ①无级别 100~150每级 ×27类型 ×3年限=486 ②永不磨损 150~160每级 ×27类型 ×3年限=162
  - ③特技(晶清诀/罗汉金钟) 100~160每级 ×6衣帽鞋饰 ×2特技 ×3年限=252 ④愤怒腰带 100~160每级 ×3年限=21
  - conditions `{"组":"无级别","类型":"男衣","等级":100,"开服年限":3}`；api_params `{"search_type":"overall_search_equip","level_min":100,"level_max":100,"kindid":18,"special_effect":1,"server_type":3,"sum_attr_without_melt":1}`
  - 种子脚本 `web/backend/init_equip.py`。⚠️装备搜索接口较敏感，易触发风控，需更慢/单独爬。

参数对照：
- 飞升=`zhuang_zhi:1`、渡劫=`zhuang_zhi:2`、化圣=`zhuang_zhi:10,20,...90`、69/109/175=`level_min/level_max`
- `sex`：1=男号，2=女号；`limit_clothes`：锦衣编码（如青花瓷=12512、潮汐帆板=14410）；`xiangrui_list`：限量坐骑，值用坐骑名字（如天使猪猪）
- 装备：`kindid`类型(男衣18/腰带20…)、`special_effect`特效(无级别1/愤怒3/永不磨损5)、`special_skill`特技(晶清诀1015/罗汉金钟2010)
- `server_type`：1/2/3 = 1年内/1到3年/3年以上（境界组与装备组的开服年限都用它）
- `server_type`：1/2/3（与 `server_map.server_age` 同码，仅境界组用）

## 6. `role_price_history` — 角色价格历史

每个 `role_query` 每天一行**全服最低价**（跨服搜索结果，非按服）。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER | 主键 |
| `query_id` | INTEGER → role_query.id | 哪个查询 |
| `run_time` | TEXT | 采集日期(年月日) |
| `price_yuan` | REAL | 全服最低价(元) |
| `serverid` / `server_name` / `area_name` | | 最低价所在服 |
| `link` / `eid` | TEXT | 商品链接 / 唯一ID |

唯一约束 `UNIQUE(query_id, run_time)`。建表/灌库脚本：`web/backend/init_roles.py`。

## 常用查询

```sql
-- 某物品某天各服最低价（带物品名/区服名）
SELECT i.name, m.area_name, m.server_name, p.price_yuan, p.link
FROM price_history p
JOIN item i ON i.id = p.item_id
LEFT JOIN server_map m ON m.serverid = p.serverid
WHERE i.name='持国巡守' AND p.run_time='2026-06-25'
GROUP BY p.serverid
ORDER BY p.price_yuan LIMIT 10;

-- 某物品全服最低价的历史走势
SELECT run_time, MIN(price_yuan) AS 全服最低
FROM price_history p JOIN item i ON i.id=p.item_id
WHERE i.name='持国巡守' GROUP BY run_time ORDER BY run_time;

-- 物品列表(含品类)
SELECT i.id, i.name, c.name AS 品类, i.type_ids
FROM item i LEFT JOIN category c ON c.id=i.category_id ORDER BY i.id;

-- 某大区下所有服务器
SELECT server_name, serverid FROM server_map WHERE area_name='上海1区';
```
